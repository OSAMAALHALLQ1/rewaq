# مصفوفة اختبارات رواق

## الأمن والعزل

| السيناريو | المتوقع |
|---|---|
| مستخدم مؤسسة A يطلب سجلًا من مؤسسة B | 403/404 ولا تسريب للبيانات |
| مدير فرع A يغير `branch_id` إلى B | رفض على الخادم وRLS |
| مستخدم دون `view_cost` | لا يرى التكلفة في UI أو API أو export |
| تعديل الدور من المتصفح فقط | لا يغير صلاحية الخادم |

## POS

| السيناريو | المتوقع |
|---|---|
| إرسال الطلب مرتين بنفس idempotency key | طلب/فاتورة/قيد واحد |
| خطأ صلاحية أثناء Offline | لا يدخل كطلب Offline |
| دفع مختلط | مجموع طرق الدفع = إجمالي الفاتورة |
| مرتجع | حركة عكسية وقيد صحيح وربط بالأصل |

## المخزون

| السيناريو | المتوقع |
|---|---|
| تحويل جزئي | كميات shipped/received/in-transit صحيحة |
| جرد Blind count | لا تظهر كمية النظام للعداد |
| فرق جرد | لا يرحّل دون اعتماد وصلاحية |
| تالف Batch | خصم من التشغيلة الصحيحة وحفظ التتبع |

## المشتريات

| السيناريو | المتوقع |
|---|---|
| استلام جزئي | يبقى PO مفتوحًا للمتبقي |
| فرق سعر فوق tolerance | يحتاج موافقة |
| فاتورة دون استلام | تمنع أو تُعلّم حسب السياسة |
| دفع جزئي | AP والرصيد والأستاذ متطابقة |

## المحاسبة

| السيناريو | المتوقع |
|---|---|
| أي ترحيل | مجموع المدين = مجموع الدائن |
| فترة مقفلة | رفض الترحيل |
| إعادة إرسال | لا قيد مكرر |
| فشل خطوة وسطية | Rollback كامل |
| عكس | قيد عكسي مرتبط بالأصل دون حذف |

## الواجهة

- RTL على سطح المكتب والهاتف.
- Keyboard focus وتسميات الحقول.
- حالات تحميل/خطأ/فارغة.
- الفلاتر والتصدير تعمل فعليًا.
- الطباعة لا تقطع البيانات المهمة.
