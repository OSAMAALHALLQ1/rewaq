# Rewaq Dashboard Design Improve

Date: 2026-05-18

## Scope

تحسين التجربة العامة لمنصة رواق بحيث تكون عملية وواضحة لصاحب مطعم أو مدير فرع، بدون افتراض معرفة محاسبية.

## Lazyweb References Used

- Wise Business dashboard: ركزت على مبدأ "الرصيد/الحالة العامة + إجراءات أولية واضحة".
- Buffer publishing dashboard: ركزت على "منشور واحد، قنوات متعددة، حالة نشر مفهومة".
- Lightspeed restaurant management: ركزت على ربط المخزون، القائمة، والفروع كمسارات تشغيلية.
- Ramp/Mercury finance operations: ركزت على بطاقات القرار والتدرج من الملخص إلى التفاصيل.

## UX Decisions Applied

1. الصفحة الرئيسية الداخلية أصبحت "اليوم في رواق" بدل لوحة أرقام فقط.
2. أضيفت إجراءات مباشرة: بيع سريع، طلب شراء، جرد سريع، منشور تسويقي.
3. أضيف قسم "ما يحتاج انتباهك" بلغة عملية: مواد تحتاج شراء، طلبات مفتوحة، تكلفة الطعام.
4. أضيف وصول واضح لكل مجالات العمل: التشغيل، المشتريات، المبيعات، القائمة، التسويق، المدفوعات.
5. تم توسيع التسويق ليعرض قنوات متعددة وحالة Node-RED بدل واجهة تجريبية مبهمة.
6. تم تبسيط الشريط الجانبي بإضافة اختصارات يومية أعلى القائمة.

## Implementation Notes

- النشر الاجتماعي يستخدم Node-RED webhook عند ضبط `NODE_RED_SOCIAL_PUBLISH_WEBHOOK_URL`.
- عند عدم وجود Node-RED، يعمل Demo fallback حتى تبقى المنصة قابلة للتجربة محليًا.
- مسار API الداخلي: `/api/node-red/social-publish`.
