# حزمة إعداد وكلاء الذكاء الاصطناعي لمشروع رواق

هذه الحزمة مصممة لكي توضع داخل **جذر مستودع رواق** ثم تُعطى إلى Codex أو Claude Code. تحتوي على:

- ملف تعليمات دائم لـ Codex: `AGENTS.md`
- ملف تعليمات دائم لـ Claude Code: `CLAUDE.md`
- ثماني Skills متخصصة في رواق داخل `.agents/skills/`
- ستة Subagents لـ Claude داخل `.claude/agents/`
- نموذج Hooks آمن داخل `.claude/settings.example.json`
- Workflow مبدئي لـ GitHub Actions
- مصفوفة اختبارات ومراجع رسمية
- ملف مهمة رئيسي جاهز للإرسال إلى Codex: `CODEX_MASTER_TASK_AR.md`

## طريقة الاستخدام السريعة

1. فك ضغط الحزمة.
2. انسخ محتويات مجلد `Rewaq_AI_Agent_Kit` إلى جذر مشروع رواق.
3. **لا تستبدل ملفات موجودة مباشرة**. ادمج المحتوى بعد المقارنة، خصوصًا `AGENTS.md` و`CLAUDE.md` وملفات CI.
4. افتح Codex من جذر المشروع.
5. أرسل له محتوى `CODEX_MASTER_TASK_AR.md` أو قل له:

```text
اقرأ README_FIRST_AR.md وCODEX_MASTER_TASK_AR.md ونفّذ الإعداد على مراحل. لا تلمس قاعدة الإنتاج، لا تكشف أسرارًا، ولا تستبدل ملفات قائمة دون نسخة احتياطية ومقارنة.
```

## الوضع الأمني الافتراضي

- GitHub MCP: قراءة فقط في البداية.
- Supabase MCP: مقيّد بمشروع واحد و`read_only=true`.
- لا تُخزن PAT أو مفاتيح Supabase أو `service_role` داخل المستودع.
- لا تُطبّق Migration على الإنتاج من دون مراجعة SQL وخطة رجوع.
- لا تسمح للوكيل بالدمج إلى الفرع الرئيسي تلقائيًا.
- استخدم Branch أو Worktree منفصل لكل مهمة.

## ترتيب التنفيذ

1. فحص المستودع والأدوات الحالية.
2. دمج ملفات التعليمات.
3. تثبيت/تفعيل Skills المحلية.
4. إعداد Playwright.
5. إعداد MCP بوضع القراءة.
6. إضافة اختبارات وCI تدريجيًا.
7. تشغيل فحص كامل وإخراج تقرير بما نجح وما فشل.

## ملاحظة مهمة

الحزمة لا تفترض أن سكربتات المشروع هي `npm`. على Codex فحص `package.json` وملف القفل أولًا واستخدام مدير الحزم الموجود بالفعل: `npm` أو `pnpm` أو `yarn` أو `bun`.
