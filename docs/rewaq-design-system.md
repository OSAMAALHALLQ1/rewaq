# نظام تصميم رواق

**الإصدار:** 1.0  
**الحالة:** أساس الواجهة الموحدة  
**اللغة الأساسية:** العربية RTL

## المبادئ

- الأزرق (`#255FDD`) يعبّر عن الإجراء الأساسي والتحديد، لا عن الزينة.
- الألوان الدلالية محجوزة للمعنى التشغيلي: نجاح، تحذير، خطر، معلومة، محاسبة وتسويق.
- الأسطح محايدة وكثيفة قابلة للقراءة؛ لا خلفيات متدرجة للجداول أو النماذج ولا ظل لكل عنصر.
- كل حالة تجمع نصاً ولوناً، وتضاف أيقونة في السياقات التي لا يكفي فيها النص وحده.
- تظل الأرقام المالية بلون النص الأساسي إلا إذا كانت الإشارة إلى هدف أو خطر أو تغير مهم.

## المكونات المشتركة

### الأزرار والحقول

- `Button default` يستخدم الأزرق 600، ويصبح 700 عند hover و800 عند الضغط.
- `outline` و`ghost` محايدان حتى يبقى الإجراء الأساسي واضحاً.
- `destructive` للحذف أو الإلغاء المدمر فقط؛ التصحيح المحاسبي والمخزني يبقى عملية عكس/تسوية موثقة، وليس حذفاً.
- `Input` و`Select` و`Textarea` تستخدم سطحاً أبيض وحداً افتراضياً واضحاً، مع focus ring أزرق قابل للتمييز.

### الشارات والحالات

| tone الحالي | المعنى | الرموز |
| --- | --- | --- |
| `default` | نشط أو صادر أو معلومة غير خطرة | `--primary-light`, `--brand-700` |
| `success` | مكتمل أو مدفوع أو متصل | `--success-bg`, `--success-text` |
| `warning` | يحتاج إجراء أو اعتماد | `--warning-bg`, `--warning-text` |
| `danger` | فشل، عجز، فرق سلبي أو إلغاء | `--danger-bg`, `--danger-text` |
| `muted` | مسودة أو حالة متوقفة/ملغاة | `--surface-tertiary`, `--text-secondary` |

عندما تُضاف حالات جديدة للمجال، تُعرّف في `StatusBadge` مع نص عربي، ولا تعتمد على اللون وحده.

### الجدول والطبقات العلوية

- رأس الجدول رمادي مزرق خفيف، لا أزرق كامل؛ الصف المحدد يستخدم `--surface-selected`.
- البطاقات العادية لها حد خفيف و`shadow-xs` فقط. Drawer وModal هما الموضعان المسموحان بـ`shadow-lg`.
- Overlay يستخدم `--overlay-bg`. يجب أن تحبس أي طبقة جديدة التركيز، وتغلق بـEsc، وتحافظ على ترتيب tab في RTL.

### الشريط الجانبي

`AppSidebar` يستخدم الأزرق الليلي. يملك الرابط النشط خلفية زرقاء شفافة وحداً في بداية السطر، بدلاً من تحويل مساحة التنقل كاملة إلى أزرق فاتح. النص غير النشط واضح لكن أقل بروزاً، والأيقونات لا تتغير إلى ألوان حالة إلا عندما يكون لذلك معنى.

## الرسوم البيانية

الترتيب الافتراضي: مبيعات أزرق، أرباح أخضر، تكلفة برتقالي، مخزون/مزامنة سماوي، مصروفات/محاسبة بنفسجي، وخسارة أو هدر مرجاني/أحمر. لا يتجاوز الرسم خمسة ألوان متى أمكن. الشبكة `--chart-grid`.

## قواعد الترحيل

1. استخدم الرموز الدلالية أو مكوّنات `ui` بدلاً من HEX داخل الصفحة.
2. لا تغيّر معاني status الحالية أثناء التحويل البصري.
3. لا تُبدّل ألوان POS أو KDS جزئياً؛ انقلها كوحدات اختبارية تحفظ وضوح السرعة والتنبيهات.
4. لا تستخدم gradient إلا في هوية محدودة أو onboarding أو تأكيد نجاح كبير، وليس خلف بيانات تشغيلية.
5. قبل اعتماد مكوّن جديد، اختبر RTL وfocus وcontrast وdesktop/tablet/mobile.
